APISCAN v1.1 – AI-enhanced OWASP API Security Scanner

Extensible scanner for the OWASP API Security Top 10 (2023). Supports Swagger/OpenAPI, multiple authentication flows (incl. OAuth2 + Bearer + mTLS + Api-key), realistic dummy requests, parallel scanning, and optional AI review.

⭐ Star on GitHub What's new Version 1.1 · MIT

What's new in v1.1

Generic Sanitizer

No customer hardcodes. Collapses duplicate path segments, normalizes /vN → /vN.00, trims trailing slashes. Toggle with --no-sanitize and refine with --rewrite "pat=>rep".

Header Overrides

Universal header model: --flow token --token, --apikey --apikey-header, --extra-header, --headers-file. Spec examples/defaults are auto‑applied too.

IDs & Samples

--ids-file to control path variables; fallback generator covers *id, code, uuid, email, date.

Method Filter

Plan/verify only certain methods: --method-filter GET POST.

Better Planning

Improved requestBody sampling order and accurate JSON detection (application/json; charset=UTF-8).

Docs

Expanded Quick Start, recipes, and troubleshooting. Medium links preserved.

Quick Start

# 1) Install (Python 3.11+ recommended)
pip install -r requirements.txt

# 2) Run with a Swagger/OpenAPI file
python apiscan.py --url https://api.example.com \
  --swagger openapi.json --flow token --token <BEARER>

# Optional: export variables template
python apiscan.py --url https://api.example.com \
  --swagger openapi.json --export_vars vars_template.yml

Bearer token

python apiscan.py --url https://api.example.com \
  --swagger openapi.json \
  --flow token --token "<JWT>" \
  --verify-plan --success-codes "200-299,304"

API key header

python apiscan.py --url https://api.example.com \
  --swagger openapi.json \
  --apikey "secret" --apikey-header x-api-key \
  --plan-only

Multiple custom headers

python apiscan.py --url https://api.example.com \
  --swagger openapi.json \
  --extra-header "x-jwt-header: secret..." \
  --extra-header "x-tenant-id: acc" \
  --verify-plan

Headers from file

# headers.json
{ "x-jwt-header": "secret...", "x-tenant-id":"acc" }

python apiscan.py --url https://api.example.com \
  --swagger openapi.json --headers-file headers.json

IDs / Path variables

# ids.json
{ "userId": "me", "orderId": 1, "Nunber": "100001" }

python apiscan.py --url https://api.example.com \
  --swagger openapi.json --ids-file ids.json

crAPI exact paths

python apiscan.py --url http://127.0.0.1:8888 \
  --swagger .\archive\crapi-openapi-spec.json \
  --flow token --token "<JWT>" \
  --no-sanitize --plan-only --verify-plan \
  --proxy 127.0.0.1:8080

Rewrite fixes

python apiscan.py --url https://api.example.com \
  --swagger openapi.json \
  --rewrite "/v2(?!\.)=>/v2.00" \
  --rewrite "/alpha/beta/alpha=>/alpha/beta" \
  --verify-plan

Only GETs

python apiscan.py --url https://api.example.com \
  --swagger openapi.json --method-filter GET \
  --plan-only --verify-plan

Troubleshooting

404 everywhere

Use --no-sanitize to avoid path normalization, or add targeted --rewrite rules. Verify paths in your proxy.

401 / 403

Expired/insufficient token or missing headers. Re‑authenticate and confirm headers via proxy.

400 / 422

Body/params invalid. Inspect requestBody in the spec and the actual request in your proxy.

Missing headers

Prefer --headers-file for environment‑specific values. Spec examples/defaults are also applied.
